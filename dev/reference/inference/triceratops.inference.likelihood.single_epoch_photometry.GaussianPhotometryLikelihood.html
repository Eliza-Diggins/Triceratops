triceratops.inference.likelihood.single_epoch_photometry.GaussianPhotometryLikelihood#

class triceratops.inference.likelihood.single_epoch_photometry.GaussianPhotometryLikelihood(model: Model, data, n_sigma_upper_limit: float = 3.0, **kwargs)[source]#

A Gaussian likelihood model for fitting single-epoch photometric data.

This likelihood model is designed to compare theoretical spectral energy distribution (SED) models against observed photometric data at a single epoch. It accounts for both detections and non-detections (upper limits) in a statistically consistent manner.

Notes

Formally, the log-likelihood is broken into two components: one for the true detections and one for the upper limits. For detected data points, we assume Gaussian errors, leading to the standard Gaussian log-likelihood formulation. For non-detections, we treat the upper limits using the cumulative distribution function (CDF) of the Gaussian distribution.

The detection component of the log-likelihood is given by:

\[\log \mathcal{L}_{\rm det} = -\frac{1}{2} \sum_{i} \left[ \frac{(F_{\rm obs, i} - F_{\rm model, i})^2}{\sigma_i^2} + \log(2 \pi \sigma_i^2) \right]\]

where \(F_{\rm obs, i}\) is the observed flux density, \(F_{\rm model, i}\) is the model-predicted flux density, and \(\sigma_i\) is the measurement error. These correspond to the freq, flux_density, and flux_density_error fields in the RadioPhotometryContainer.

The upper limit component of the log-likelihood is given by:

\[\log \mathcal{L}_{\rm upper} = \sum_{j} \log \left[ \Phi\left( \frac{F_{\rm limit, j} - F_{\rm model, j}}{\sigma_j} \right) \right],\]

where \(F_{\rm limit, j}\) is the upper limit flux density, \(F_{\rm model, j}\) is the model-predicted flux density, \(\sigma_j\) is the noise level, and \(\Phi\) is the CDF of the standard normal distribution:

\[\Phi(x) = \frac{1}{2} \left[ 1 + {\rm erf}\left( \frac{x}{\sqrt{2}} \right) \right].\]

Missing Errors on Non-Detections:

It is common for non-detections to lack explicit error estimates. In such cases, we infer the noise level from the provided upper limit using a specified number of standard deviations, \(n_\sigma\). Specifically, if the upper limit is given as \(F_{\rm limit}\), we estimate the noise level as:

\[\sigma = \frac{F_{\rm limit}}{n_\sigma}.\]

By default, \(n_\sigma = 3\), but this can be adjusted via the n_sigma_upper_limit parameter during initialization. If the flux_density_error field is provided for non-detections, any np.nan fields will be filled with the predicted uncertainty from this calculation.

__init__(model: Model, data, n_sigma_upper_limit: float = 3.0, **kwargs)[source]#

Initialize the GaussianPhotometryLikelihood object.

This performs:

  1. Compatibility validation between model and data.

  2. Storage of the raw model and data container.

  3. One-time preprocessing of the data into numerical form suitable for fast likelihood evaluation.

Parameters:

  • model (Model) – The astrophysical model to be evaluated. This must be a subclass of Model and be compatible with the likelihood class (see COMPATIBLE_MODELS). This defines the forward model being fit against.

  • data – The data container object holding the observational data. This must be of a type compatible with the likelihood class (see COMPATIBLE_DATA_CONTAINERS). This defines the observations being compared against.

  • n_sigma_upper_limit (float, optional) – The number of standard deviations above the noise level to consider as the upper limit for non-detections. Default is 3.0. See the class notes (GaussianPhotometryLikelihood) for more details on how upper limits are treated in the likelihood.

  • **kwargs – Additional keyword arguments passed to the data processing method. These are unused.

Raises:

TypeError: – If the provided model or data are not compatible with this likelihood.

Notes

Upon initialization, the likelihood object processes the input data via the _process_input_data() method. The results are cached in self._data for efficient reuse during likelihood evaluations. All data is coerced into consistent base units during this step.
